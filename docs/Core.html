14.2 Core

 (require pollen/core) package: pollen

这些函数会自动导入到每个 Pollen 源文件中（意思是，就好像它们已包含在您的 "pollen.rkt" 中一样）。

14.2.1 Metas

每个元表中唯一自动定义的键是 'here-path ，它持有源文件的绝对路径。例如，你可以用(select-from-metas 'here-path metas)来检索这个值。 有关元数据的完整介绍，请参阅 Inserting metas 。

syntax

(define-meta name value)

将 value 添加到当前文档的元数据中，使用 name 作为键。

您可以使用 (select-from-metas name metas) 检索元值——甚至在你定义它的同一个文档中。

14.2.2 Splicing

syntax

(@ arg ...)

拼接标签表示一个列表应该被合并到它的包含表达式中。拼接标签为'@。

Examples:
> (module splicer pollen/markup
  '(div "one" (@ "two" "three") "four"))
> (require 'splicer)
> doc

'(root (div "one" "two" "three" "four"))

当你想在只能返回一个 X 表达式的情况下返回一个列表时，拼接标签很有用。例如， Tag functions 只能返回一个 X-表达式。但是如果我们把 X-表达式列表包裹在一个拼接标签中，它们的行为就像一个单一的 X-表达式。之后，Pollen 会将列表元素合并到周围的表达式中（如上所示）。

Examples:
> (require pollen/tag)
; wrong: function returns a list of X-expressions
> (define-tag-function (multi attrs elems)
    '("foo" "bar"))
; right: function returns a list of X-expressions
; as elements inside a splicing tag
> (define-tag-function (multi2 attrs elems)
    '(@ "foo" "bar"))

尽管拼接标签在外观上与 unquote-splicing 的 @ 的缩写符号相同，并且具有相似的目的，但它不是一回事。拼接标签不是变量——它只是 Pollen 在生成输出时特别对待的一个符号。

syntax

(when/splice condition pollen-args)

如果 condition 为真，则将 pollen-args 放入文档中。在模板文件中，通常这样调用：

◊when/splice[condition]{The text to insert.}

插入的文本可以包含其自己的嵌套 Pollen 命令。

when/splice 比 when 更方便，因为 when 只会使用大括号之间的最后一个参数。相比之下， when/splice 将花括号之间的所有内容视为一个块。

syntax

(for/splice (for-clause ...) body-or-break ... body)

syntax

(for*/splice (for-clause ...) body-or-break ... body)

与 for/list 和 for*/list 类似，但将生成的列表拼接到文档中。

Added in version 1.4 of package pollen.

14.2.3 Data helpers

从 Pollen 源文件中检索数据的函数。这些不是唯一的选择——当然，您可以使用任何常用的 Racket 函数。

procedure

(get-doc doc-source) → (or/c txexpr? string?)

  doc-source : (or/c pagenode? pathish?)
从 doc-source 中检索 doc 导出，它可以是路径、路径字符串或可以解析为源路径的页面节点。如果无法解析 doc-source ，则引发错误。

如果 doc-source 是相对路径或页面节点，则将其视为相对于 current-project-root 。如果这不是您想要的，您需要将其显式转换为完整路径（例如，使用 path->complete-path 或 ->complete-path ）。

如果 setup:main-export 被重写了一个项目特定的值，那么它将被检索出来。

procedure

(get-metas meta-source) → hash?

  meta-source : (or/c pagenode? pathish?)
从 meta-source 检索 metas 导出，它可以是路径、路径字符串或可以解析为源路径的页面节点。如果无法解析 meta-source ，则引发错误。

如果 meta-source 是相对路径或页面节点，则将其视为相对于 current-project-root 。如果这不是您想要的，您需要将其显式转换为完整路径（例如，使用 path->complete-path 或 ->complete-path ）。

如果 setup:meta-export 被重写了一个项目特定的值，那么它将被检索出来。

procedure

(select key value-source) → (or/c #f xexpr?)

  key : symbolish?
  value-source : (or/c hash? txexpr? pagenode? pathish?)

procedure

(select* key value-source) → (or/c #f (listof xexpr?))

  key : symbolish?
  value-source : (or/c hash? txexpr? pagenode? pathish?)
在 value-source 中查找 key 的匹配项。 value-source 可以是 1) metas 的哈希表，2) 表示 doc 的标记 X 表达式，或 3) 标识源文件的页面节点或路径，该源文件提供 metas 和 doc 。在这种情况下，首先在 metas 中查找 key （使用 select-from-metas ），然后在 doc 中（使用 select-from-doc ） .

用 select ，你会得到第一个结果；用 select* ，你会得到所有结果。

In both cases, you get #f if there are no matches.

注意，如果 value-source 是一个相对路径或 pagenode，它将被视为相对于 current-project-root 。如果这不是你想要的，你需要明确地将其转换为完整路径（例如，用 path->complete-path 或 ->complete-path ）。

Examples:
> (module nut-butters pollen/markup
  '(div (question "Flavor?")
    (answer "Cashew") (answer "Almond")))
; Import doc from 'nut-butters submodule
> (require 'nut-butters)
> (select 'question  doc)

"Flavor?"

> (select 'answer  doc)

"Cashew"

> (select* 'answer  doc)

'("Cashew" "Almond")

> (select 'nonexistent-key doc)

#f

> (select* 'nonexistent-key doc)

#f

procedure

(select-from-doc key doc-source) → (or/c #f (listof xexpr?))

  key : symbolish?
  doc-source : (or/c txexpr? pagenodeish? pathish?)
在 doc-source 中查找 key 的值。 doc-source 参数可以是 1) 表示 doc 的标记 X 表达式或 2) 标识提供 doc 的源文件的页面节点或源路径。如果 key 的值不存在，你会得到 #f 。

注意，如果 doc-source 是一个相对路径或 pagenode，它将被视为相对于 current-project-root 。如果这不是你想要的，你需要明确地将其转换为完整路径（例如，用 path->complete-path 或 ->complete-path ）。

Examples:
> (module gelato pollen/markup
  '(div (question "Flavor?")
    (answer "Nocciola") (answer "Pistachio")))
; Import doc from 'gelato submodule
> (require 'gelato)
> (select-from-doc 'question  doc)

'("Flavor?")

> ('answer . select-from-doc . doc)

'("Nocciola" "Pistachio")

> (select-from-doc 'nonexistent-key doc)

#f

procedure

(select-from-metas key meta-source) → any/c

  key : symbolish?
  meta-source : (or/c hash? pagenodeish? pathish?)
在 meta-source 中查找 key 的值。 meta-source 参数可以是 1) 表示 metas 的哈希表或 2) 标识提供 metas 的源文件的页面节点或源路径。如果 key 的值不存在，你会得到 #f 。

注意，如果 meta-source 是一个相对路径或 pagenode，它将被视为与 current-project-root 相对。如果这不是你想要的，你需要明确地将其转换为完整路径（例如，用 path->complete-path 或 ->complete-path ）。

Examples:
> (define metas (hash 'template "sub.xml.pp" 'target "print"))
> (select-from-metas 'template  metas)

"sub.xml.pp"

> ('target . select-from-metas . metas)

"print"

> (select-from-metas 'nonexistent-key metas)

#f

14.2.4 Parameters

parameter

(current-metas) → (or/c #f hash?)

(current-metas val) → void?
  val : (or/c #f hash?)
 = #f
保存当前 Pollen 源的 metas 。在标签函数中，您可以在函数体中引用 (current-metas) ，而不是将 metas 作为参数传递。同样，如果您的标签函数调用其他标签函数，它们都可以调用 (current-metas) 而不是传递值。

(current-metas) 也会在模板中工作，持有当前被渲染到模板中的源的 metas 。

默认值是 #f 。这意味着没有 metas 值可用。明智地处理这种情况是您的责任。

Added in version 1.4 of package pollen.
