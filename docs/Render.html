14.6 Render

 (require pollen/render) package: pollen

Rendering 是 Pollen 源文件转换为输出的方式。

procedure

(render source-path [template-path]) → (or/c string? bytes?)

  source-path : complete-path?
  template-path : (or/c #f complete-path?) = #f
渲染 source-path 。渲染行为取决于源文件的类型（有关详细信息，请参阅(part ("(lib pollen/scribblings/pollen.scrbl)" "File_formats"))）：

在没有模板的情况下渲染 pollen/pre 文件。

使用模板渲染 pollen/markup 或 pollen/markdown 文件。如果没有使用 template-path 指定模板，Pollen 会尝试使用 get-template-for 找到模板。

请注意，使用模板进行渲染在 eval 中使用 include-template 。对于复杂的页面，第一次可能会很慢。缓存用于使后续请求更快。

对于那些对使用 eval 感到恐慌的人，请不要这样做。正如 include-template 的作者已经建议的那样，“如果你坚持动态主义”——是的，我坚持——“总是会有 eval 的。”

procedure

(render-to-file source-path    
  [template-path    
  output-path]) → void?
  source-path : complete-path?
  template-path : (or/c #f complete-path?) = #f
  output-path : (or/c #f complete-path?) = #f
与 render 类似，但将文件保存到 output-path ，覆盖已经存在的所有内容。如果没有提供 output-path ，它是使用 ->output-path 从 source-path 派生的。

procedure

(render-to-file-if-needed source-path    
  [template-path    
  output-path]) → void?
  source-path : complete-path?
  template-path : (or/c #f complete-path?) = #f
  output-path : (or/c #f complete-path?) = #f
与 render-to-file 类似，但仅当存在以下条件之一时才会进行渲染：

  1. output-path 处不存在文件。 （因此，强制渲染特定 output-path 的一种简单方法是删除它。）

  2. 自上次通过 render 以来， source-path 、 template-path 或关联的 "pollen.rkt" 发生了变化。

  3. 渲染缓存已停用。

如果这些条件都不存在， output-path 被认为是最新的，并跳过渲染。

procedure

(render-batch source-path ...) → void?

  source-path : pathish?
一次性渲染多个 source-paths 。如果您的 source-paths 依赖于一组通用模板，这可能比 (for-each render source-paths) 更快。模板可能有自己的需要编译的源文件。如果您使用 render ，模板将被重复（并且不必要地）重新编译。而如果你使用 render-batch ，每个模板只会被编译一次。

procedure

(render-pagenodes pt-or-pt-source) → void?

  pt-or-pt-source : (or/c pathish? pagetree?)
使用 pt-or-pt-source ，使用 render-batch 渲染该页面树中的页面节点。

请注意， pt-or-pt-source 严格作为一个要渲染的文件列表使用的，就像一个批处理文件。它并不被用作渲染文件的导航页树。

procedure

(get-template-for source-path) → (or/c #f complete-path?)

  source-path : complete-path?
查找 source-path 的模板文件，优先级如下：

  1. 如果 source-path 的 metas 有template 的键，则使用该键的值，例如—

    ◊(define-meta template "my-template.html")

    如果你的项目有多个输出目标，你可以提供一个模板列表，扩展名与当前输出目标匹配的模板将被自动选择，例如—

    ◊(define-meta template (list "my-template.html" "my-template.txt" "my-template.pdf"))

  2. 如果此键不存在，或引用不存在的文件，则查找名称为 template.[output extension] 的默认模板。意思是，如果 source-path 是intro.html.pm，那么输出路径就是 intro.html ，所以默认模板是template.html。在与源文件相同的目录中查找此默认模板，然后在连续的父目录中向上搜索。 （推论：项目根目录中的默认模板将应用于项目中的所有文件，除非在子目录中被覆盖。）

  3. I如果此文件不存在，请使用后备模板作为最后的手段。 （另见 (part ("(lib pollen/scribblings/pollen.scrbl)" "tutorial-2" "Templates")).)

当需要一个模板，但缺少 template-path 参数时（例如，在 render 或 render-to-file 中），会调用这个函数。
