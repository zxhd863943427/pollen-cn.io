14.4 File

 (require pollen/file) package: pollen

一个实用模块，提供了处理 Pollen 源文件和输出文件的功能。在普通使用中，你可能不需要这些。但如果你想对 Pollen 进行更精细的处理，它们就在这里。

Pollen 可以处理六种源文件：

  • Preprocessor ，文件扩展名为 .pp

  • Markup ，文件扩展名为 .pm

  • Markdown ，文件扩展名为 .pmd

  • Null ，文件扩展名为 .p

  • Scribble ，文件扩展名为 .scrbl

  • Pagetree ，文件扩展名为 .ptree。这是唯一不生成输出文件的源类型。

这个模块中的函数依赖于 pollen/setup 中指定的文件扩展。这些扩展名可以在一个项目中被覆盖–见 How to override setup values 。

对于每种 Pollen 源文件，对应的输出文件名是通过从源文件名中去掉扩展名而得到的。所以预处理器源文件 "default.css.pp" 会变成 "default.css" 。 (另见 (part "Saving___naming_your_source_file") ，如果这一点没有印象的话。)

Scribble 文件的工作方式不同–相应的输出文件是源文件，但扩展名为 "html" ，而不是 "scrbl" 。所以 "pollen.scrbl" 会变成 "pollen.html" 。

关于 Pollen 的文件模型的更多信息，请参见 File formats 。

  p : pathish?

(get-scribble-source p) → (or/c #f path?)

  p : pathish?
查找将产生输出路径 p 的现有源路径。

综合 get-source 将按以下顺序检查源格式： get-markup-source 、 get-markdown-source 、 get-preproc-source 、get-null-source 和 get-scribble-source。

当然，特定类型的变体将只返回一个指定类型的源文件。

在所有情况下，如果没有相应的源文件，则返回 #f 。

procedure

(->output-path p) → path?

  p : pathish?
将一个源路径 p 转换为其对应的输出路径。这个函数只是为一个文件生成一个路径–它并不询问该文件是否存在。

如果 p 有 poly 输出类型 ，那么 ->output-path 将使用 current-poly-target 作为输出路径扩展。

否则，这个函数没有特定类型的变体，因为 Pollen 源文件的输出路径是 由其名称决定的 。

Examples:
> (->output-path "main.css.pp")

#<path:main.css>

> (->output-path "default.html.pm")

#<path:default.html>

> (->output-path "index.html.p")

#<path:index.html>

> (->output-path "file.scrbl")

#<path:file.html>
