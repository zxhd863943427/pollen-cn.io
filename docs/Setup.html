14.7 Setup

 (require pollen/setup) package: pollen

14.7.1 How to override setup values

可以通过在 “poly.rkt” 源文件中重写以下值来更改这些值：

  1. 在这个文件中，创建一个名为 setup 的子模块。

  2. 在这个子模块中，使用 define 来创建一个与 pollen/setup 中的变量同名但没有 setup: 前缀的变量。

  3. 给它分配你喜欢的任何值。

  4. 根据需要重复。

  5. (别忘了 provide setup 子模块中的变量。)

当 Pollen 运行时，这些定义将取代 pollen/setup 中的定义。

例如，假设您希望将每个 Pollen 源文件的主要导出称为 van-halen 而不是 doc， Pollen 标记文件的扩展名是 .rock 而不是 .pm，命令字符是 🎸 而不是 ◊ 。你的 "pollen.rkt" 会看起来像这样：

"pollen.rkt"
#lang racket/base
 
;; ...通常的定义和标签函数...
 
(module setup racket/base
  (provide (all-defined-out))
  (define main-export 'van-halen)
  (define markup-source-ext 'rock)
  (define command-char #\🎸))

当然，您可以通过从 "pollen.rkt" 中删除这些定义的值来恢复默认值。

每个setup:name 函数都会解析该变量的当前值：它将从 setup 子模块返回值（如果此处定义了 name ），否则它将返回默认值（可直接从default-name 中获得）。例如， default-command-char 将始终为 ◊ ，但在上面的示例中， (setup:command-char) 将返回 🎸 。

procedure

(setup:cache-watchlist) → (listof (or/c path?path-string?))

value

default-cache-watchlist : (listof (or/c path?path-string?))

 = '()
缓存（= 渲染缓存 + 编译缓存，统称）在项目服务器会话期间监视的额外文件列表。如果监视列表中的文件之一发生更改，则缓存无效（就像 "pollen.rkt" 更改时一样）。

如果缓存在监视列表中找不到某个文件，则不会出现错误。该文件将被忽略。因此，为避免意外行为，最好的策略是使用完整路径（或路径字符串）。生成本地文件完整路径的一种方法是使用 define-runtime-path 。另一种方法是，如果您正在使用已作为包的一部分安装的模块，则使用 resolve-module-path：

"pollen.rkt"
(module+ setup
  (provide (all-defined-out))
  (require racket/runtime-path syntax/modresolve)
  (define-runtime-path my-local-mod "my-module.rkt")
  (define my-installed-mod (resolve-module-path 'package/my-other-module))
  (define cache-watchlist (list my-local-mod my-installed-mod)))

Added in version 1.4 of package pollen.

procedure

(setup:envvar-watchlist) → (listof string?)

value

default-envvar-watchlist : (listof string?) = '()

缓存键中使用的额外环境变量列表。将为环境变量的每个不同值维护单独的缓存。 The POLLEN environment variable 总是被使用，不管这个值是如何设置的。

环境变量的名称和值都不区分大小写，因此 "PUB" 和 "pub" 和 "pUb" 都被视为相同的变量。

Added in version 2.1 of package pollen.

procedure

(setup:index-pages) → (listof string?)

value

default-index-pages : (listof string?) = '("index.html")

项目服务器将用作目录默认页面的字符串列表，按优先级排序。对命令行渲染操作没有影响。对您的实时 Web 服务器也没有影响（通常这是您需要在 .htaccess 配置文件中进行的设置）。 但是使用此设置，您可以模拟实时服务器的行为，以便内部 index-page URL 工作正常。

procedure

(setup:trim-whitespace?) → boolean?

value

default-trim-whitespace? : boolean? = #t

控制 Pollen 源阅读器是否从 doc 导出的开头修剪空白的谓词。如果您将 Pollen 用作另一种编程语言的预处理器并且想要准确地保留前导空格，则可以将其设置为 #false。

procedure

(setup:external-renderer)

 → (or/c (list/c module-path? symbol?) #f)

value

default-external-renderer

 : (or/c (list/c module-path? symbol?) #f)
 = #f
一个模块路径和标识符（适合与 dynamic-require 一起使用），提供一个函数供 Pollen 在渲染 project server 所需的文件时调用而不是 render 或者在运行 raco pollen render 时。该函数必须接受与 render-to-file 相同的参数，并且应该以 string 或 byte string 的形式返回最终输出。 Pollen 将始终为您将此返回值写入输出文件。

设置此值使您可以完全控制（并负责）Pollen 如何将编译的 doc 和 metas 从源文件转换为最终输出。您的渲染器应该能够处理 Pollen 的任何 source formats 或 utility formats。 Pollen 的 render 函数的操作不受设置此值的影响，因此您的渲染器可以将其用作后备。

Added in version 3.2 of package pollen.
